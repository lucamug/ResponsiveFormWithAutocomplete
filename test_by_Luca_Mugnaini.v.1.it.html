Metodo di pagamento

Note
  • The city field is disabled unless the user select a country
  • Based on the country selected, the city suggestion is filtered and return only cities of that country
  • An error message appear if no cities are found by the public api
  • The shipping cost is update upon country selection
  • The page is responsive
  • The layout is based on Skeleton
  • The page use the public api Ajax Autocomplete Cities List by Geobytes
  • There is a simple client side validation based on HTML5 specifications
  • It has been tested only on Google Chrome (PC and mobile)
  • A map appear upon city selection. From an UX perspective I would probably ask to remove it. But when I notice that the public api could return also latitude and longitude I could not resist. I like maps
  • Supports some level of graceful degradation, for example it works with Javascript turned off
  • Done some refactoring to achieve a more testable Javescrit with Jasmine
  • Find other comments in common.js
Possible Improvements
  • Run User Testing (think aloud) to verify the simplicity and clarity of the interface
